# Iteración: Visitas — 1.0.18 (numero, calendar drag, fix clone, edit completadas, validaciones)
Fecha: 2026-04-28
Estado: esperando revisión

## Requerimiento

Cinco cambios en módulo Visitas para release 1.0.18:

1. **ID correlativo (`numero`)** visible al abrir cualquier visita en el header del form.
2. **Calendar lateral como filtro principal** en `/visitas/lista`: reemplaza inputs Desde/Hasta. Soporta click simple (1 día) y arrastre (rango). Default al cargar: semana actual.
3. **Bug de fechas al clonar**: el reporte indica que la fecha queda desfasada. Causa: `toLocal/fromLocal` parsean datetime-local strings de forma inconsistente cross-browser. Fix: usar `new Date(year, month-1, day, hour, minute)` que es siempre local.
4. **Editar visitas completadas** con permiso `visitas.editar`. Hoy el form no tiene gate de permiso. Sumar gate; las completadas se vuelven editables si tenés el permiso (los inputs respetan el permiso, no el estado). El trigger 013 sigue bloqueando transiciones FUERA de completada para no super_admin.
5. **Validación bidireccional estado ↔ técnico**: hoy solo se valida "si hay técnicos → estado debe ser X". Sumar el inverso "si estado es X → debe haber al menos un técnico".

## Criterios de aceptación

- [ ] Header del form muestra `#{{numero}}` al abrir una visita existente. En "nueva" no se muestra.
- [ ] `/visitas/lista` al cargar: la semana actual está marcada en el calendar (lunes a domingo). La lista filtra por esa semana sin que el user toque nada.
- [ ] Inputs Desde/Hasta NO están en el formulario de filtros.
- [ ] Click en un día del calendar → selecciona ese día (rango = 1 día).
- [ ] Mouse down en día A + drag a día B + mouse up → rango A→B (visualmente highlighteado).
- [ ] Persistir el rango aunque cambies de mes con los botones « ».
- [ ] Modal de clonar: configurar fecha 30/04 14:00, clonar, abrir la nueva visita → fecha es 30/04 14:00 exactamente (no 11:00 ni 17:00).
- [ ] Visita completada con `visitas.editar`: campos editables, guardar OK.
- [ ] Sin `visitas.editar`: banner read-only, todos los inputs disabled, botón Guardar disabled.
- [ ] Guardar `agendado_con_tecnico` con 0 técnicos: error explícito.
- [ ] Guardar `en_cola` con técnicos: error explícito (validación inversa, ya existente).

## Notas visuales / referencias

- Banner read-only: caja amarillo claro (`bg-amber-50 dark:bg-amber-900/20`) con icono 🔒 y texto "Modo solo lectura. No tenés permiso para editar visitas."
